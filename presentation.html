The Problem

Traditional DeFi Lending is Broken

Existing protocols send funds directly to borrowers — there is no way to enforce how the money is used, who bears the loss, or when the lender gets paid back.

🔓

No Fund Control

Borrowers receive funds directly. Lenders have zero visibility into how capital is deployed or risked.

📊

Opaque Risk

Credit scores are on-chain or absent entirely. Private borrower data cannot be verified without disclosure.

⚡

Binary Liquidation

Losses hit lender principal immediately. There is no staged waterfall to absorb shock before principal is affected.

03 / 09

Our Solution

Controlled Vault Protocol

Funds never leave the protocol. A PDA vault holds all capital, borrowers execute only whitelisted strategies, and profit/loss is settled trustlessly on-chain.

Control

PDA-Secured Vault

All capital held in a Program-Derived Address token account. Neither borrower nor lender can unilaterally withdraw funds.

Intelligence

AI + ZK Risk Layer

DeepSeek AI evaluates credit risk in plain English. snarkjs ZK proofs verify private factors without exposing raw data.

Protection

Loss Waterfall

Losses hit strategy reinvest pool → borrower collateral → lender principal. Lender is always the last to be affected.

Core Mechanics

Profit Split & Loss Waterfall

🟢 Profit Day — 5% / 95% Split

// Every profitable cycle: Vault NAV gain × 5% → Lender Profit Lock Pool Vault NAV gain × 95% → Strategy Reinvest Pool // Lender profit pool is ISOLATED — // borrower cannot trade with it // Exit condition: locked_profit >= target_profit AND vault_nav >= principal + locked

🔴 Loss Day — Waterfall Absorption

1
Strategy Reinvest Pool
First buffer — absorbs losses from borrower's trading activity
2
Borrower Collateral
Second buffer — when reinvest pool is depleted
3
Liquidation Trigger
If vault_nav < 120% of loan — strategy stopped, lender protected
✓
Lender Principal — Always Last
Profit lock pool and principal protection are NEVER deducted from trading losses

06 / 09

Solana Technologies

Built on the Full Solana Stack

On-Chain

Anchor Framework 0.32

7 typed instructions with full account validation, discriminator arrays, PDA seed derivation, and custom error codes. IDL exported for frontend type-safe client.

Program Derived Addresses (PDA)

Vault account is a PDA — deterministically derived from borrower pubkey. Neither party controls the private key. Funds are only movable via program instructions.

SPL Token Program

USDC SPL token accounts are used for all vault deposits, profit accounting, and withdrawals. Token transfers gated by on-chain state checks.

Off-Chain / Frontend

@solana/wallet-adapter

Phantom wallet integration with React hooks. Wallet public key used as borrower identity, PDA seed source, and transaction signer.

@solana/web3.js + @coral-xyz/anchor

Frontend calls Anchor program methods via typed client. Server-side RPC calls use undici ProxyAgent for network compatibility.

Solana Devnet

All on-chain interactions (vault state reads, instruction submissions) target Solana Devnet. Program deployed and verifiable on-chain.

AI + ZK Dual Layer

Smart Risk — Private by Default

🤖 DeepSeek AI Risk Engine

The system sends borrower risk parameters to the DeepSeek Chat API and receives a structured JSON risk report. When the API key is unavailable, a local rule-based fallback engine runs automatically.

🔐 snarkjs ZK Privacy Proofs

Sensitive borrower data is verified using Groth16 zero-knowledge proofs. Only the proof hash is stored on-chain. The raw inputs never touch the blockchain.

CreditVault Account State

// Key fields in CreditVault PDA borrower: Pubkey lender: Pubkey amount: u64 // loan principal collateral: u64 // borrower deposit vault_nav: u64 // real-time NAV lender_profit_share:u64 // locked profit strategy_pool: u64 // reinvest pool credit_score: u8 // AI score 0-100 proof_hash: [u8; 32] // ZK proof hash status: VaultStatus

Design Principles

Off-chain AI + ZK, on-chain state — Risk scoring and ZK proof generation happen off-chain. Only results (score, proof hash) stored on-chain.
PDA-controlled vault — Funds held in a PDA token account; neither borrower nor lender can access without program logic.
Loss waterfall — Strategy pool absorbs losses first, then collateral. Lender principal protected until liquidation.
Lender profit lock — Only 5% of profit locked per round; rest compounds in strategy reinvest pool for continued yield.
